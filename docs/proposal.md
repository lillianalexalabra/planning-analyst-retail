## Reflection

This posting is directly relevant to this course because it requires the exact skills we have built across every unit: SQL for querying and analyzing sales and inventory data, Python for building automated data pipelines, and dashboards for reporting KPIs to leadership teams. The Planning Analyst role at SKIMS centers on demand forecasting, inventory management, and sell-through analysis — all problems that are solved with the data engineering and analytics workflow this class teaches end-to-end. To prove I can do this job, I am building a pipeline that pulls US Census Bureau monthly retail sales data via API, loads it into Snowflake, transforms it through a dbt star schema, and surfaces seasonal demand trends and inventory risk signals for the shapewear and intimates category in a deployed Streamlit dashboard — the same kind of KPI reporting the role expects. This project transfers naturally to adjacent roles such as inventory analyst, merchandise planner, or demand planning analyst at any fashion or retail company, since the skills, schema design, and analytical questions are consistent across the industry.
